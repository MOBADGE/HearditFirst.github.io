**Memory Market Sees Slight Shift in DDR5 Prices but Contract Rates Stay Firm** Market analyst Trendforce reports a noticeable price drop in retail DDR5 RAM kits across the US, Europe, and China. This adjustment is attributed mainly to consumer-driven factors rather than a fundamental shift in demand. While some online retailers show fluctuating prices, contract prices from major memory suppliers remain stable, supported by multi-year agreements with key clients. The broader memory market is influenced by variables such as AI industry spending, ongoing supply constraints, and evolving technology demands. For PC gamers and builders, this means while some DDR5 kits might become slightly more affordable, a significant price correction is not yet guaranteed.

**Serious Security Concern: Popular JavaScript Library Axios Targeted by Supply Chain Attack** Axios, a widely used JavaScript library with millions of weekly downloads, was compromised by attackers who published malicious versions containing a remote access trojan (RAT). Google has attributed the attack to a North Korea-linked financially motivated group identified as UNC1069 or CryptoCore. The trojan allows attackers to gain remote control over affected systems. Although the official Axios library code remained unaffected, the compromised versions were available briefly through npm, potentially impacting projects that installed dependencies during that window. Security firms advise developers to audit systems, monitor for suspicious activity, and rotate credentials to mitigate any risks stemming from this sophisticated supply chain attack.

**Experiment Shows It’s Not Feasible to Run a PC Without System RAM** YouTube channel PortalRunner explored whether it is possible to operate a PC without traditional system RAM amid the ongoing memory price crisis. Extensive testing found that drastically reducing DRAM led to system failures or severely degraded performance, including failed game launches and low memory throughput. Attempts to substitute system RAM with graphics card VRAM also encountered stability issues. The final experiment involved modding a BIOS chip to use CPU cache as memory, allowing only very basic output such as a minimal Snake game clone. The findings underscore that conventional DRAM remains essential for functional, practical computing, leaving no viable alternative to mitigate current memory shortages through hardware workarounds.

**Getting the Assessor Matrix Material in Arc Raiders** The recently introduced Assessor Matrix material in Arc Raiders is required for advanced crafting projects such as the High-Gain Antenna, but it is difficult to obtain. Players must engage with the Close Scrutiny event condition, where Assessor drop pods fall onto the map and can be breached for loot. These pods are well-defended by hostile Arc enemies and contested by other players. Successful collection depends on tactics like using stealth equipment (Photoelectric Cloak), smoke grenades, and strong weaponry to claim the rare Assessor Matrix drops. Collaboration with other players can help control threats but risks sharing the loot. This challenge adds complexity to resource gathering in the game’s evolving meta.

**Nvidia’s Dynamic Multi Frame Generation Faces Issues with Frame Rate Limiters** Nvidia’s new Dynamic Multi Frame Generation (DMFG), part of the DLSS 4.5 update available to RTX 50-series owners, enhances frame rates by dynamically generating multiple AI-assisted frames. However, when a frame rate cap is applied through Nvidia's app, DMFG locks into fixed generation modes, which can increase input latency (PCL) and reduce smoothness. Testing showed latency rising from around 30 milliseconds without limits to 50-60 milliseconds with a 138 fps cap. This may be due to a bug or a design constraint in the technology. Users relying on precise frame caps for tear-free gameplay may need to disable DMFG or stick with standard Multi Frame Generation until Nvidia addresses the issue.

**Intel’s BOT Software May Overstate CPU Performance Compared to AMD, Says Geekbench Creator** Primate Labs, makers of Geekbench benchmarking software, criticize Intel’s Boost Optimization Technology (BOT) for generating misleading benchmark results. BOT rearranges and vectorizes code to optimize Intel CPUs but supports only a limited set of applications. This selective optimization can make Intel processors appear faster against AMD in tests than they would perform during typical real-world use. Primate Labs’ tests showed significant code changes, including instruction count reductions through vectorization, which Intel’s documentation does not fully disclose. The team also noted BOT induces startup delays and requires application-level optimization by Intel. This calls into question direct benchmark comparisons when BOT is enabled.
